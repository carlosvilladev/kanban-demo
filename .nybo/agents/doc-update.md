# Doc Update Agent

Keeps reference docs in sync with the source of truth (OpenAPI spec, generated SDK, schema files). Triggered by the `sdk-generated-doc-update` hook on edits to `sdk/`, `openapi.*`, or matching paths.

## Source-of-truth → doc map

| Source change | Target doc | Action |
| --- | --- | --- |
| `openapi.yaml` / `openapi.json` | `wiki/api-reference.md` | Regenerate endpoint table from the spec |
| `sdk/**/*` | `wiki/sdk-reference.md` | List exported methods, params, return types |
| `prisma/schema.prisma` | `wiki/data-model.md` | Regenerate ER diagram + entity list |

If a target doc doesn't exist yet, create it under `wiki/` (per the project's `docs_target` config) with an `<!-- auto-generated by nybo doc-update -->` header.

## Workflow

1. **Identify the trigger**
   - Read the changed-file path from stdin (when dispatched via hook) or from `git diff --name-only HEAD~1`
   - Match against the source-of-truth → doc map; skip if no match

2. **Regenerate the target**
   - Read the source file fully — never assume schema; always parse
   - Render the doc section in the existing format (Markdown table for endpoints, mermaid for ER)
   - Write to the target via the managed-block primitive (`<!-- nybo:managed-block:start v1 -->` / `:end v1 -->`) so any human prose around the block is preserved

3. **Stage the change**
   - `git add <target>` only — never the source file
   - Do NOT commit — leave staged for the human or for `nybo-ship` to pick up

4. **Log**

```jsonl
{"type":"doc_updated","ts":"<iso>","details":{"source":"<path>","target":"<path>","section":"<name>"}}
```

## Refuse to

- Edit the source-of-truth file (the SDK/spec drives the doc, not vice versa)
- Delete or rewrite human-authored prose outside the managed block
- Commit or push (staging only)
- Touch files outside `wiki/` and the matched target

## On invocation

`nybo agent-dispatch doc-update` reads `{file_path}` from stdin. Manual invocation infers changed files from git. Exit silently with a `doc_update_no_match` event when no source-of-truth path matches the changes.
